# Spring Boot Notes (Lectures 48-82)

## 1) Quick Review (Why Spring Boot)
- Spring (core framework) is powerful but usually needs more setup and boilerplate.
- Spring Boot reduces setup using:
  - Auto-configuration
  - Starter dependencies
  - Embedded server (Tomcat by default)
  - Opinionated defaults
- Result: faster development, less XML/manual config, easy production-ready app start.

## 7) Layered Design (Structuring Thoughts)
Flow:
1. Client (Browser/Postman) sends HTTP request
2. Controller receives request
3. Service executes business logic
4. (Later) Repository talks to database
5. Response returned to client

Current app has Controller + Service + Model (in-memory list), no DB yet.

## 7.1 How the API Actually Works Internally
1. Embedded Tomcat receives request on configured port.
2. Spring MVC `DispatcherServlet` routes request to matching controller method.
3. `@RequestBody` JSON is converted to Java object using Jackson.
4. Service method executes business rules.
5. Return value is converted back to JSON and sent with HTTP status.

Why this matters:
- You write plain Java methods, Spring handles routing + conversion + HTTP plumbing.

### 8.0 Request Mapping Notes (Important)
- `@RequestMapping("/api")` at class level sets common prefix.
- Method-level mappings append route + verb:
  - `@GetMapping`
  - `@PostMapping`
  - `@PutMapping`
  - `@DeleteMapping`
- Equivalent old style:

      @RequestMapping(value = "/public/categories", method = RequestMethod.GET)

Use the composed annotations (`@GetMapping`, etc.) for cleaner code.

### 8.1 GET All Categories
Controller:

    @GetMapping("/public/categories")
    public ResponseEntity<List<Category>> getAllCategories(){
        List<Category> categories = categoryService.getAllCategories();
        return new ResponseEntity<>(categories, HttpStatus.OK);
    }

### 8.2 POST Create Category
Controller:

    @PostMapping("/public/categories")
    public ResponseEntity<String> createCategory(@RequestBody Category category){
        categoryService.createCategory(category);
        return new ResponseEntity<>("Category added successfully", HttpStatus.CREATED);
    }

Service technique used:
- Maintain in-memory list
- Auto-increment id using nextId

    private List<Category> categories = new ArrayList<>();
    private Long nextId = 1L;

    public void createCategory(Category category) {
        category.setCategoryId(nextId++);
        categories.add(category);
    }

### 8.3 DELETE Category
Controller catches service exception and maps to HTTP response.

    @DeleteMapping("/admin/categories/{categoryId}")
    public ResponseEntity<String> deleteCategory(@PathVariable Long categoryId){
        try {
            String status = categoryService.deleteCategory(categoryId);
            return ResponseEntity.status(HttpStatus.OK).body(status);
        } catch (ResponseStatusException e){
            return new ResponseEntity<>(e.getReason(), e.getStatusCode());
        }
    }

Service uses stream lookup + ResponseStatusException when not found.

### 8.4 PUT Update Category

    @PutMapping("/public/categories/{categoryId}")
    public ResponseEntity<String> updateCategory(@RequestBody Category category,
                                                 @PathVariable Long categoryId){
        try{
            Category savedCategory = categoryService.updateCategory(category, categoryId);
            return new ResponseEntity<>("Category with category id: " + categoryId, HttpStatus.OK);
        } catch (ResponseStatusException e){
            return new ResponseEntity<>(e.getReason(), e.getStatusCode());
        }
    }

      ### 8.5 @PathVariable Quick Rule
      - URL segment value is bound by name:

        @DeleteMapping("/admin/categories/{categoryId}")
        public ResponseEntity<String> deleteCategory(@PathVariable Long categoryId)

      - Path placeholder and parameter name should match unless explicitly mapped:

        @PathVariable("categoryId") Long id

## 11) Important Notes and Improvements
- Current storage is in-memory; data resets on restart.
- Next milestone should introduce:
  - JPA/Hibernate
  - Entity mapping
  - Repository layer
  - Database config in application.properties
- API path consistency improvement:
  - Create currently uses /public/categories
  - The design slide also shows /api/admin/category for create
  - Pick one convention and keep all write endpoints consistent.
- Response consistency improvement:
  - `PUT` currently returns only message text.
  - Prefer returning updated object or a DTO for predictable client behavior.
- Thread-safety caveat:
  - In-memory `ArrayList` + mutable counter are fine for learning, not production-safe for concurrent writes.

## 12) Interview/Revision One-Liners
- Spring Boot starters: dependency bundles that simplify setup.
- Auto-configuration: Boot configures beans based on classpath and settings.
- Embedded server: app runs standalone (no external Tomcat required).
- ResponseEntity: explicit control over status code, headers, body.
- @PathVariable: binds dynamic URL segment to method argument.
- ORM: maps objects to relational tables, reducing boilerplate SQL.

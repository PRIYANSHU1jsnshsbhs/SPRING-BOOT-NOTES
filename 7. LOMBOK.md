## 99) What Is Lombok?

### Definition
Lombok is a Java library that reduces boilerplate code by generating common code at compile time using annotations.

### Why Lombok Is Used
- Avoids writing repetitive getters/setters/constructors.
- Makes model/entity classes shorter and easier to read.
- Speeds up development.
- Improves maintainability by reducing manual repetitive code.

### Typical Boilerplate Lombok Replaces
- Getters and setters
- `toString()`
- `equals()` and `hashCode()`
- Constructors
- Builder pattern
- Logger field declaration

### How It Works (High Level)
- You add Lombok annotations to classes/fields.
- Annotation processing runs during compilation.
- Lombok generates bytecode-equivalent methods.
- Generated methods are available to your code even though you do not see them written manually.

### Dependency Setup

Maven:
```xml
<dependency>
	<groupId>org.projectlombok</groupId>
	<artifactId>lombok</artifactId>
	<version>1.18.36</version>
	<scope>provided</scope>
</dependency>
```

Gradle:
```gradle
compileOnly 'org.projectlombok:lombok:1.18.36'
annotationProcessor 'org.projectlombok:lombok:1.18.36'
testCompileOnly 'org.projectlombok:lombok:1.18.36'
testAnnotationProcessor 'org.projectlombok:lombok:1.18.36'
```

## 100) Lombok Annotations You Should Know

## Core Data/Class Annotations

### `@Getter` and `@Setter`
- Generates getter/setter methods.
- Can be applied at field level or class level.

Example:
```java
import lombok.Getter;
import lombok.Setter;

@Getter
@Setter
public class User {
	private Long id;
	private String name;
}
```

### `@ToString`
- Generates `toString()` method.
- Useful for logging/debugging objects.

```java
import lombok.ToString;

@ToString
public class Product {
	private Long id;
	private String title;
}
```

### `@EqualsAndHashCode`
- Generates `equals()` and `hashCode()`.
- Important for collections (`Set`, `Map`) and identity comparisons.

### `@NoArgsConstructor`, `@AllArgsConstructor`, `@RequiredArgsConstructor`
- `@NoArgsConstructor`: no-arg constructor.
- `@AllArgsConstructor`: constructor with all fields.
- `@RequiredArgsConstructor`: constructor for `final` and `@NonNull` fields.

```java
import lombok.AllArgsConstructor;
import lombok.NoArgsConstructor;

@NoArgsConstructor
@AllArgsConstructor
public class Employee {
	private Long id;
	private String name;
}
```

### `@Data`
- Shortcut annotation.
- Includes:
  - `@Getter`
  - `@Setter`
  - `@ToString`
  - `@EqualsAndHashCode`
  - `@RequiredArgsConstructor`

Use it for simple DTO/POJO classes.

### `@Value`
- Immutable variant of `@Data`.
- Class becomes effectively immutable (final fields, no setters).

## Builder and Utility Annotations

### `@Builder`
- Implements builder pattern automatically.
- Useful when constructors have many parameters.

```java
import lombok.Builder;
import lombok.ToString;

@Builder
@ToString
public class Order {
	private Long id;
	private String item;
	private int quantity;
}

// Usage:
// Order order = Order.builder().id(1L).item("Laptop").quantity(2).build();
```

### `@Singular`
- Used with `@Builder` for collections.
- Adds single-item and full-collection builder methods.

### `@Builder.Default`
- Keeps a default value when object is built and field is not explicitly set.

### `@NonNull`
- Adds null-check for parameters/fields.
- Throws `NullPointerException` with a clear message.

### `@SneakyThrows`
- Allows throwing checked exceptions without declaring `throws`.
- Use carefully; can reduce readability if overused.

### `@Cleanup`
- Ensures resource cleanup (similar to try-finally behavior).
- Rarely used now because `try-with-resources` is standard.

### `@Slf4j` (and other logger annotations)
- Auto-generates logger field.

```java
import lombok.extern.slf4j.Slf4j;

@Slf4j
public class PaymentService {
	public void process() {
		log.info("Processing payment");
	}
}
```

## 101) Dark Mode in IntelliJ (Quick Notes)

This topic is IDE-related (not Lombok-specific), but useful for setup.

### Enable Dark Theme
1. Open IntelliJ IDEA.
2. Go to `File > Settings` (Windows/Linux) or `IntelliJ IDEA > Settings` (macOS).
3. Navigate to `Appearance & Behavior > Appearance`.
4. Set `Theme` to `Darcula` (or another dark theme).
5. Apply and close.

### Optional: Install More Themes
1. Go to `Settings > Plugins`.
2. Search `Theme`.
3. Install preferred theme plugin.
4. Restart IDE if prompted.

## 102) Lombok Plugin and Annotation Processing

Lombok usually needs both:
- Lombok dependency in project
- Annotation processing enabled in IDE

### A) Install Lombok Plugin in IntelliJ
1. Open `Settings > Plugins`.
2. Search for `Lombok`.
3. Install plugin by JetBrains/community listing.
4. Restart IntelliJ.

### B) Enable Annotation Processing
1. Open `Settings`.
2. Go to `Build, Execution, Deployment > Compiler > Annotation Processors`.
3. Check `Enable annotation processing`.
4. Apply changes.

### C) Verify Dependency
Make sure `pom.xml` or `build.gradle` includes Lombok correctly.

### D) Reimport/Rebuild
- Maven: Reimport project, then run `mvn clean compile`.
- Gradle: Refresh Gradle project, then run `gradle clean build`.

## 103) Lombok in Action

### Example Without Lombok (Verbose)
```java
public class Student {
	private Long id;
	private String name;

	public Student() {}

	public Student(Long id, String name) {
		this.id = id;
		this.name = name;
	}

	public Long getId() { return id; }
	public void setId(Long id) { this.id = id; }

	public String getName() { return name; }
	public void setName(String name) { this.name = name; }

	@Override
	public String toString() {
		return "Student{" + "id=" + id + ", name='" + name + '\'' + '}';
	}
}
```

### Same Class With Lombok (Clean)
```java
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

@Data
@NoArgsConstructor
@AllArgsConstructor
public class Student {
	private Long id;
	private String name;
}
```

### Practical Spring Boot DTO Example
```java
import lombok.Builder;
import lombok.Data;

@Data
@Builder
public class JobResponse {
	private Long id;
	private String title;
	private String company;
	private boolean active;
}
```

Usage:
```java
JobResponse response = JobResponse.builder()
		.id(10L)
		.title("Backend Engineer")
		.company("EmbarkX")
		.active(true)
		.build();
```

## 104) Resolving "Lombok Not Working"

If Lombok annotations show errors (red underline) or generated methods are "missing", follow this checklist.

### Quick Fix Checklist
1. Confirm Lombok dependency exists in build file.
2. Confirm IntelliJ Lombok plugin is installed and enabled.
3. Enable annotation processing in IntelliJ.
4. Reimport Maven/Gradle project.
5. Run clean build (`mvn clean compile` or `gradle clean build`).
6. Invalidate caches and restart IntelliJ.
7. Ensure JDK/project SDK is correctly configured.
8. Update Lombok to a recent compatible version.

### Common Root Causes
- Missing annotation processing
- Lombok plugin not installed
- Wrong dependency scope/config
- IDE cache corruption
- Version mismatch between JDK and Lombok/IDE

### IntelliJ-Specific Recovery Sequence
1. Install/verify Lombok plugin.
2. Enable annotation processing.
3. Reload project.
4. `Build > Rebuild Project`.
5. `File > Invalidate Caches` and restart if needed.

## Best Practices While Using Lombok
- Use `@Data` for DTOs, but be careful using it blindly on JPA entities.
- Prefer explicit annotations (`@Getter`, `@Setter`, etc.) when you need fine control.
- Avoid overusing `@SneakyThrows`.
- Keep team conventions consistent (same Lombok usage style).
- Ensure CI build also includes annotation processing support.

## Lombok + JPA Caution
- JPA entities often need a no-args constructor.
- Be careful with `@EqualsAndHashCode` on entities (ID lifecycle can cause issues).
- Avoid including lazy-loaded relationships in `toString()` to prevent performance issues and recursion.

## One-Line Summary
Lombok reduces Java boilerplate with compile-time code generation, but it requires proper dependency setup, IDE plugin support, and annotation processing to work reliably.

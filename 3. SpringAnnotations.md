# Spring Notes: Annotations, Component Scan, @Value, @Autowired, @Qualifier

## 41) Introduction to Annotations

### What is an annotation?
- Annotation = metadata added to Java code.
- It does not directly execute business logic.
- Frameworks (like Spring) read annotations and apply behavior.

### Quick Java example
    @Override
    public String toString() {
        return "...";
    }

### Common Spring annotations
- `@Component`
- `@Autowired`
- `@Qualifier`
- `@Value`
- `@Repository`
- `@Service`
- `@Controller`
- `@RequestMapping`
- `@SpringBootApplication`

## 42) Understanding Component and ComponentScan

### Component
- A component is a Java class managed by Spring IoC container.
- You mark it using `@Component` (or specialized stereotypes like `@Service`, `@Repository`, `@Controller`).

### Component Scan
- Spring scans package paths to detect classes with component annotations.
- Auto-registers them as beans.

### XML way to enable scan
    <context:component-scan base-package="com.example.componentscan"/>

### Annotation/Java config way
    @Configuration
    @ComponentScan(basePackages = "com.example.autowired.annotation")
    public class AppConfig {}

## 43) Hands-on: Component and Component Scan (Without Annotations)

This part is XML-centric bean setup (no `@Component` required).

### Folder structure you need
    src/main/java/com/example/demo/
      App.java
      Car.java
      Engine.java
    src/main/resources/
      applicationContext.xml

### Code

#### `Engine.java`
    package com.example.demo;

    public class Engine {
        private String type;

        public void setType(String type) {
            this.type = type;
        }

        @Override
        public String toString() {
            return "Engine{type='" + type + "'}";
        }
    }

#### `Car.java`
    package com.example.demo;

    public class Car {
        private Engine engine;

        public void setEngine(Engine engine) {
            this.engine = engine;
        }

        public void show() {
            System.out.println("Car -> " + engine);
        }
    }

#### `applicationContext.xml`
    <?xml version="1.0" encoding="UTF-8"?>
    <beans xmlns="http://www.springframework.org/schema/beans"
           xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
           xsi:schemaLocation="
            http://www.springframework.org/schema/beans
            https://www.springframework.org/schema/beans/spring-beans.xsd">

        <bean id="engine" class="com.example.demo.Engine">
            <property name="type" value="V8"/>
        </bean>

        <bean id="car" class="com.example.demo.Car">
            <property name="engine" ref="engine"/>
        </bean>

    </beans>

#### `App.java`
    package com.example.demo;

    import org.springframework.context.ApplicationContext;
    import org.springframework.context.support.ClassPathXmlApplicationContext;

    public class App {
        public static void main(String[] args) {
            ApplicationContext ctx = new ClassPathXmlApplicationContext("applicationContext.xml");
            Car car = ctx.getBean("car", Car.class);
            car.show();
        }
    }

### Implementation steps
1. Create POJOs (`Car`, `Engine`) with setter methods.
2. Define beans in XML.
3. Wire dependency using `ref`.
4. Load context in `App` and fetch bean.
5. Run and verify output.

## 44) `@Value` Annotation

### Purpose
- Inject primitive/String/system property/SpEL values into bean fields or setters.

### Examples from your project
- In `com.example.componentscan.Employee` and `com.example.autowired.annotation.Employee`:

    @Value("Hello")
    private String firstName;

    @Value("${java.home}")
    private String lastName;

    @Value("#{4*4}")
    private double salary;

### Usage patterns
- Constant: `@Value("Hello")`
- Property placeholder: `@Value("${key}")`
- Spring Expression Language (SpEL): `@Value("#{...}")`

## 45) Transition from XML to Annotations in Spring

## Before (XML-heavy)
- Define every bean in XML.
- Wire using `<property ref="...">`.
- Enable component scan via XML when needed.

## After (Annotation-driven)
- Add `@Component` to classes.
- Use `@Autowired` for dependency injection.
- Use `@Configuration + @ComponentScan` for bootstrap.
- Keep XML minimal or remove it.

### Migration strategy (practical)
1. Start with one package/module.
2. Add `@Component` on candidate beans.
3. Replace XML bean wiring with `@Autowired`.
4. Introduce Java config class (`@Configuration`).
5. Validate app startup.
6. Remove redundant XML entries.

## 46) `@Autowired` Annotation

### Purpose
- Inject dependency automatically by type.

### Injection styles
- Field injection
- Setter injection
- Constructor injection (recommended for required dependencies)

### Example from your project (`Manager`)
    @Component
    public class Manager {
        @Autowired
        @Qualifier("employee")
        private Employee employee;
    }

### Constructor version (recommended style)
    @Component
    public class Manager {
        private final Employee employee;

        @Autowired
        public Manager(@Qualifier("employee") Employee employee) {
            this.employee = employee;
        }
    }

### Common issue
- If multiple beans of same type exist, Spring throws `NoUniqueBeanDefinitionException`.
- Fix with `@Qualifier`.

---

## 47) `@Qualifier` Annotation

### Purpose
- Select a specific bean when more than one candidate of same type exists.

### Example
    @Autowired
    @Qualifier("employee")
    private Employee employee;

- Here bean name `employee` maps to `@Component("employee")`.

### Best practice
- Keep bean names meaningful.
- Prefer constructor injection + qualifier in parameters.

## End-to-end implementation checklist

1. Add dependency in `pom.xml` (Spring Context).
2. Create POJO classes.
3. Decide wiring approach:
   - XML wiring, or
   - Annotation wiring.
4. If annotations:
   - add `@Component` on beans,
   - add `@Autowired` dependencies,
   - resolve conflicts via `@Qualifier`,
   - inject constants/properties with `@Value`,
   - create `@Configuration` + `@ComponentScan`.
5. Create main class and load context.
6. Run and validate output.

---

## Interview-ready one-liners

- `@Component`: Registers class as Spring-managed bean.
- `@ComponentScan`: Tells Spring where to scan for components.
- `@Autowired`: Injects dependency by type.
- `@Qualifier`: Chooses exact bean among multiple candidates.
- `@Value`: Injects literal/property/SpEL value.

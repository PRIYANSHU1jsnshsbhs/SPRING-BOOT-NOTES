# Understanding Data and Databases

Source: Faisal Memon (EmbarkX)

## 1) What Is a Database?
- A database is a place where data related to users and products is stored.
- Applications use databases to save and retrieve information.

### Basic Flow
- User interacts with application.
- Application stores or fetches data from the database.

## 2) Types of Databases
- Relational databases
- Non-relational databases

## 3) What Is a DBMS?
- DBMS stands for Database Management System.
- It is software that helps users and applications interact with a database.
- It supports creating, reading, updating, and deleting data.

### Examples of DBMS
- MySQL
- Oracle Database
- SQL Server
- MongoDB
- Cassandra

### Types of DBMS
- Relational Database Management System (RDBMS)
- NoSQL Database Management System

## 4) Relational Database Concepts

### Core Terminologies
- Table: Collection of related data in rows and columns.
- Column: A field/attribute in a table.
- Row: One record in a table.
- Primary Key: Unique identifier for each row.
- Foreign Key: Column that links one table to another.
- Index: Improves data retrieval speed.
- Query: A request to read or modify data.

### Row Synonyms
- Row = Tuple = Record

## 5) Overview of SQL
- SQL stands for Structured Query Language.
- SQL is used to retrieve, manage, and update data in a database.
- Database data is accessed through SQL queries.

### Types of SQL Queries
- DQL (Data Query Language)
- DML (Data Manipulation Language)
- DDL (Data Definition Language)
- DCL (Data Control Language)
- TCL (Transaction Control Language)

## 6) What Is ORM?
- ORM stands for Object-Relational Mapping.
- ORM maps classes in code to tables in a database.
- Class attributes are mapped to table columns.
- Objects are mapped to rows in tables.

### ORM Example (Customer)

Class representation:
- `id: Integer`
- `first_name: String`
- `last_name: String`

Possible objects:
- Customer 1: id=1, first_name="John", last_name="Trump"
- Customer 2: id=2, first_name="Stacy", last_name="Keiber"
- Customer 3: id=3, first_name="Mark", last_name="Dsouza"

Database table representation:

| id | first_name | last_name |
|---|---|---|
| 1 | John  | Trump  |
| 2 | Stacy | Keiber |
| 3 | Mark  | Dsouza |

### Why ORM Is Useful
- Reduces manual SQL writing for common operations.
- Can auto-create tables from classes (depending on framework/config).
- Lets developers focus more on application logic.
- Reduces human errors in repetitive SQL tasks.
- Handles translation between application objects and SQL/database structure.
